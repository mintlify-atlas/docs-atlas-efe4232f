llmake can be installed globally via npm, bun, or pnpm, or run directly without installation using npx/bunx/pnpx.

## Global Installation

Install llmake globally to use the `llmake` command from anywhere:

<CodeGroup>

```bash npm
npm i -g llmake
```

```bash bun
bun add -g llmake
```

```bash pnpm
pnpm add -g llmake
```

</CodeGroup>

After installation, verify it works:

```bash
llmake --version
# 0.1.1
```

## Run Without Installing

If you prefer not to install globally, you can run llmake directly:

<CodeGroup>

```bash npx
npx llmake
```

```bash bunx
bunx llmake
```

```bash pnpx
pnpx llmake
```

</CodeGroup>

<Info>
Running without installation is great for CI/CD pipelines or trying llmake before committing to a global install.
</Info>

## Building from Source

For development or to use the latest unreleased features, you can build llmake from source:

```bash
# Clone the repository
git clone https://github.com/CyrusNuevoDia/llmake
cd llmake

# Install dependencies
bun install

# Build standalone binary
bun run build:bin
# Creates bin/llmake
```

The compiled binary will be available at `bin/llmake` and can be moved to any location in your `PATH`.

<Note>
Building from source requires [Bun](https://bun.sh) to be installed on your system.
</Note>

## Verifying Installation

Once installed, verify llmake can find your shell environment:

```bash
llmake --help
```

You should see the full help output:

```
llmake 0.1.1

Usage:
  llmake                     Run all tasks with changes
  llmake <task>              Run specific task if changed
  llmake --force [task]      Run regardless of hash state
  llmake --dry-run [task]    Show what would run
  llmake --status            Show per-task change status
  llmake --init              Write starter llmake.jsonc
  llmake --config <path>     Use specific config file
  llmake --help              Print this help
  llmake --version           Print version
```

## Shell Environment

llmake runs commands in a login shell to ensure proper environment loading. This means your runner CLI needs to be available in your PATH as configured in your shell profile (`.zshrc`, `.bashrc`, etc.).

<Warning>
If llmake can't find your runner command, check that it's properly installed and available in your shell's PATH.
</Warning>
