## Why llmake?

No more regenerating docs when nothing changed. No more forgetting to update generated files when sources change.

llmake tracks your source files with SHA-256 hashes, compares against a lockfile, and only runs your LLM when changes are detected. It's `make`, but for LLM-powered code generation.

## Key Features

<CardGroup cols={2}>
  <Card title="Content-Addressed Hashing" icon="fingerprint">
    SHA-256 hashing for incremental builds. Only regenerate when source files actually change.
  </Card>
  <Card title="Declarative Configuration" icon="file-code">
    Configure tasks in JSON, JSONC, TOML, or TypeScript. Define prompts, sources, and runners.
  </Card>
  <Card title="Runner-Agnostic" icon="plug">
    Works with Claude Code, OpenAI, Ollama, or any LLM CLI that accepts prompts.
  </Card>
  <Card title="Lockfile Tracking" icon="lock">
    Commit `.llmake.lock` for team-wide consistency and CI builds.
  </Card>
  <Card title="Flexible Glob Patterns" icon="filter">
    Match source files with glob patterns and exclude specific files.
  </Card>
  <Card title="Multiple CLI Modes" icon="terminal">
    Force run, dry run, status check, and more. Full control over task execution.
  </Card>
</CardGroup>

## Quick Example

```bash
# Install globally
npm i -g llmake

# Initialize config
llmake --init

# Run all tasks with changes
llmake

# Check what's stale
llmake --status
```
